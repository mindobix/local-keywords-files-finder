Keywords Files Finder

Find any file on your machine by what's inside it. Crawls your filesystem, reads spreadsheet headers and document text, and surfaces matches by keyword — in real time, entirely offline.

▶ Open App localhost:5173 ↗ GitHub
Features
🗂️
Filesystem Crawler
Watches your home directory continuously. New and changed files are indexed within seconds via Chokidar.
🏷️
Keyword Matching
Each keyword has path hints and header hints — match by filename or by column names inside the file.
📂
Category Sidebar
Files are organized into collapsible keyword categories. A NEW section highlights recently discovered files.
🔬
File Preview
Click any file to open a paginated data grid with sortable columns. Search within columns inline.
⚡
Live Updates via SSE
The UI stays in sync over Server-Sent Events. Files appear as they are indexed — no refresh needed.
💾
Backup & Restore
Export your keyword taxonomy as a portable JSON file. Restore on any machine or share with a team.
🌗
Dark / Light Theme
Toggle themes from the top bar. Preference is persisted across sessions in localStorage.
🔒
100% Private
No telemetry. No cloud. Only metadata (path, column names) is stored — file contents never leave your machine.

How to Run
⚙️
Getting Started
1
Requirements
Node.js 18 or later and npm must be installed.
2
Install dependencies
npm install npm run setup-hooks # installs the git pre-commit version-bump hook
3
Start the app
npm run dev # or: npm run dev:fresh — wipes the index and starts clean
Starts three processes concurrently:
Process Port Role
Indexer 3002 File crawler + control API
API Server 3001 REST + SSE for the UI
Vite Dev Server 5173 React frontend
4
Open in your browser
http://localhost:5173
The crawler starts watching your home directory immediately. Add keywords in the Settings panel and matches will appear in the sidebar as files are indexed.
5
Optional — watch a custom path
WATCH_PATH=/Volumes/Data npm run dev # override the default home-directory watch path
Built locally — no internet required
